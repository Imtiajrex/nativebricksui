# Input Component

The `Input` component is a versatile and accessible text input field designed for React Native applications. It seamlessly integrates with the `InputContainer` to provide a customizable and consistent user experience, including support for states like `focused` and `invalid`.

## Props

The `Input` component combines the properties of:

1. `TextInputProps` from React Native.
2. `InputContainerProps` from the `InputContainer` utility.

### Key Props

- `leading`: Optional. A React Node displayed before the input field (e.g., icons).
- `trailing`: Optional. A React Node displayed after the input field (e.g., buttons).
- `state`: Controls the state of the input. Common states are `default`, `focused`, `valid`, and `invalid`.
- `placeholder`: Text placeholder within the input field.
- `className`: Additional Tailwind classes for custom styling.

## Usage

Hereâ€™s how to use the `Input` component:

```tsx
import { Input, useColor } from '@nativebricks/core';
import React from 'react';
import { View } from 'react-native';
import { User } from 'lucide-react-native';

export default function InputPage() {
  return (
    <View className="container">
      {/* Input with a leading icon */}
      <Input
        placeholder="Ex: John Doe"
        leading={
          <View className="p-2">
            <User size={24} color={useColor('muted-foreground')} />
          </View>
        }
      />

      {/* Input with invalid state */}
      <Input placeholder="Ex: example@gmail.com" state="invalid" />

      {/* Simple Input */}
      <Input />
    </View>
  );
}
```

1. **Input with Leading Icon**:

   - Demonstrates the use of the `leading` prop to include an icon inside the input field.

2. **Input with Invalid State**:

   - Shows how to visually indicate invalid input by setting the `state` prop to `invalid`.

3. **Simple Input**:
   - A basic usage example of the `Input` component with minimal configuration.

## Styling

The `container` prop allows you to apply container styles using Tailwind classes.
And the `className` prop allows you to apply custom styles to the input field itself
For instance:

```tsx
<Input containerClassName="bg-muted" className="text-sm" />
```

## Integration Notes

- Ensure the `Input` is wrapped inside an `InputContainer` (this is automatically handled when using the `Input` component).
- The component uses `useFocus` to manage focus states internally, making it easier to handle `onFocus` and `onBlur` events.
